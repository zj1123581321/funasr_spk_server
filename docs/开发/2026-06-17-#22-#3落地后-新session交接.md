### 关键设计点(避免下个 session 误解)
- **#22 只堵"错开到达"**:同 file_hash 先后上传,第二份排队期间第一份已转完写缓存 → 开工前
  命中秒返回。**"几乎同时到达"(多 worker 并发取出双双 miss)堵不住** = 已知局限,pool 有富余
  槽兜底。完整并发去重 = TODOS #22(自愈 index + 连接迁移 + canonical-id,blast radius 大,本轮不做)。
- **#22 不建 index**:正是为避开 prior learning `inflight-dedup-coupling`(去重 index 要在 6+
  终态点维护,漏一处→脏 index→轮询命中死 task_id)。修法一只查缓存,零 index。
- **#3 只做接收端**:`classify_error` 靠字符串兜底(底层 funasr/qwen3 仍抛裸 Exception+中文 message)。
  **发送端类型化异常(`EngineInitError` 等)= TODOS #3 完整版,碰两引擎核心,本轮不做**。
