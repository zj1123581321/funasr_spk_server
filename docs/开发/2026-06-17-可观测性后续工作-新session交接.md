## 三、关键上下文 / 环境坑(务必先读)

- **严格 TDD**：先红 → 绿 → commit,最小单位提交,不积累多改动一次提交。
- **不开 PR**：本项目独立维护,完成后直接 push / merge main(单人 main 工作流),不要说"PR 标题/等 review"。
- **改 FunASR 路径必跑 parity**(schemas/database/task_manager/websocket_handler/funasr_transcriber)：
  `FUNASR_RUN_INTEGRATION=1 FUNASR_DEFAULT_ENGINE=funasr venv/bin/python -m pytest tests/integration/test_parity_funasr_semantic.py tests/integration/test_funasr_server_websocket_e2e.py`
  或便利脚本 `bash scripts/run_checks.sh --parity`。
- **pytest 用 `venv/bin/python -m pytest`**(venv pytest binary shebang 漂移)。
- **dev 机 .env 污染**：纯净环境 6 个 config 测试会因 dev .env 误挂;unit 有 hermetic fixture 兜底,parity 要显式 `FUNASR_DEFAULT_ENGINE=funasr`;ort_parity 在 Mac 上必挂(正常)。
- **PM2 命令走 Bash 工具**(不要走 ctx_execute,会污染 TMPDIR 导致 daemon ENOENT)。
- **部署**(macOS only,不走 docker):prod `~/Production/funasr_spk_server`(8767,PM2),dev 本目录(8867,前台)。github SSH 偶发抖,prod 拉不到时可从同机 dev 仓库本地 `git fetch <dev路径> main`。
- **架构权威**:项目根 `CLAUDE.md`(引擎/config/队列/可观测性各章节)。
